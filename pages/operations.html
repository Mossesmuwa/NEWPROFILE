OP-004 Red Team
Telnet Exploitation Lab
Like reading someone's postcard — the password was right there in plain text
Learning exercise using tutorial guidance + Metasploitable 2 (intentionally vulnerable VM)
Connected to Metasploitable 2 via Telnet from Kali Linux, then used Wireshark to capture and analyse the TCP stream. Credential recovery was straightforward — the point of the exercise was understanding why unencrypted protocols are dangerous, not the difficulty of exploiting them.
Telnet Wireshark Kali Linux Metasploitable 2
Difficulty
~2 hours · Flip for findings
GitHub
// Key findings — OP-004
Telnet transmits everything — usernames, passwords, commands — in plain unencrypted text visible to anyone on the network
Default credentials (msfadmin/msfadmin) still active — credential hygiene is the most neglected security control
Wireshark captured 1,359 bytes of conversation including full authentication sequence in one capture session
Remediation: Disable Telnet entirely. Replace with SSH. Change all default credentials immediately. Network segmentation reduces blast radius if unencrypted protocols must remain.

OP-003 Offensive
Kali Linux Penetration Test
Scanned the target, found the open doors, walked right in
Learning exercise using tutorial guidance + Metasploitable 2 (intentionally vulnerable VM)
Full penetration test workflow on Metasploitable 2 — Nmap recon mapped 13 open ports and services, then identified and exploited the vsftpd 2.3.4 backdoor (CVE-2011-2523) using Netcat to gain a root shell. The goal was practising the methodology: recon → identify → research → exploit → document.
Nmap CVE-2011-2523 Netcat Kali Linux
Difficulty
~4 hours · Flip for findings
Writeup
// Key findings — OP-003
vsftpd 2.3.4 backdoor exploited — root shell obtained. Known vulnerability from 2011 still present on unpatched system
13 open ports discovered — FTP, SSH, Telnet, HTTP, SMB, MySQL all exposed to network
Linux kernel 2.6.24 — over a decade out of date, multiple critical CVEs available
Nmap OS fingerprinting confirmed Linux 2.6.x — useful for targeting specific exploits
Remediation: Keep all software patched. Disable services not in use. Firewall rules — whitelist only what's needed. Principle of least privilege throughout.

OP-002 Web Security
Web Application Security Lab
Found the kinds of holes that attackers exploit in real websites every day
Learning exercise using DVWA (Deliberately Vulnerable Web Application) — designed for exactly this
Tested DVWA against OWASP Top 10 vulnerabilities. Each category was studied from both sides — understanding how the attack works before learning how to prevent it. XSS and SQL injection were the most impactful findings.
DVWA Burp Suite OWASP Top 10 SQLMap
Difficulty
~5 hours · Flip for findings
// Key findings — OP-002
SQL Injection — extracted full database contents by injecting SQL through input fields
Stored XSS — injected script executed for every user viewing the affected page
CSRF — forged requests performed actions as authenticated users without their knowledge
Remediation: Prepared statements for SQL. Input validation and output encoding for XSS. CSRF tokens for state-changing requests. Content Security Policy headers.

OP-001 IT Systems
Windows Server & Active Directory Lab
Built a fake company network from scratch — then tested how easy it was to attack
Built following tutorial guidance, then independently tested attack paths against my own setup
Set up Windows Server 2019 as a domain controller, created OUs and user accounts, applied GPO hardening, then switched hats and tested the configuration — AS-REP Roasting, BloodHound enumeration, and privilege escalation attempts. Both sides of the same coin.
Windows Server 2019 Active Directory GPO BloodHound Impacket
Difficulty
~8 hours · Flip for findings
// Key findings — OP-001
AS-REP Roasting — 3 accounts had Kerberos pre-authentication disabled, password hashes extracted offline
Overprivileged accounts — standard user accounts in Domain Admins group without justification
Weak password policy — minimum 6 characters, no complexity requirement, no lockout threshold
Remediation: Enable Kerberos pre-auth for all accounts. Enforce least privilege. Minimum 12-character passwords with complexity. Account lockout after 5 failed attempts. Regular AD auditing with BloodHound.
